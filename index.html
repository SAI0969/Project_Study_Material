1. Computer Architecture vs. Computer Organization

Understanding the fundamental line of demarcation between the conceptual design space and the real-world operational hardware layout implementation.

Definition Computer Architecture

Computer Architecture is the logical design of a computer system that defines how the system behaves and what functions it performs. It specifies the programmer-visible aspects of the computer.

Keywords:

Logical design Programmer's view Functionality ISA (Instruction Set Architecture)

Definition Computer Organization

Computer Organization is the way the hardware components are arranged and interconnected to implement the architecture efficiently.

Keywords:

Physical implementation Hardware design Internal operation Performance

Relationship Between Architecture and Organization

  • Architecture tells what the computer should do.
  • Organization tells how the computer performs those tasks.
  • Core Axiom: Many distinct organizations can implement the identical target architecture.

Example: Two laptops may have the same Intel Core i5 architecture but completely different cache sizes, clock speeds, and memory technologies. Both run the exact same software because their architecture matches perfectly.

Internal CPU Registers & Core Memory Hierarchy

Registers are very small, ultra-fast local memory workspaces constructed inside the CPU block:

  • Program Counter (PC): Stores the address of the next execution instruction.
  • Instruction Register (IR): Holds the currently executing instruction code payload.
  • Accumulator (AC): Stores immediate arithmetic computing results.
  • Memory Address Register (MAR): Holds the memory location target address actively being read or written.
  • Memory Data Register (MDR): Holds the data payload transferred from or going into memory.
Memory Hierarchy (Fastest → Slowest):
Registers → Cache → RAM → SSD/HDD

*As speed decreases, total capacity increases, and cost per GB drops.

The Fetch-Decode-Execute Cycle & Foundations

The Von Neumann Architecture establishes that instructions and data are stored in the same unified memory space. The CPU continuously runs a 3-step cycle loop:

  1. Fetch Instruction: Pulls code from memory address tracked by PC into the IR.
  2. Decode Instruction: The Control Unit parses opcode fields to identify commands.
  3. Execute Instruction: ALU handles logic calculations, updates flags or registers.

2. Fundamentals of Computer Architecture

Detailed study of basic block diagrams, core performance parameters, subsystems, and execution configurations required for exam reference.

Characteristics of a Computer (Exam 2-Mark Guide)

Speed: Processes millions of operations per second.
Accuracy: Error-free calculations unless faulty data is input.
Automation: Executes full task sequences without human intervention.
Diligence: Free from monetary fatigue, weakness, or concentration gaps.
Reliability: Gives consistent outputs for identical inputs over time.
Versatility: Performs widely different tasks (e.g., math, media processing).
Large Storage: Stores bulk archives securely across memory systems.

System Word Length Parameters

The total count of bits that a CPU processor can manipulate and act upon in a single clock operation cycle:

  • 32-bit architecture: Processes 32 bits of information simultaneously at once.
  • 64-bit architecture: Processes 64 bits of information simultaneously at once, allowing access to significantly larger RAM spaces.

Processor Clock Speed Metrics

The operating frequency tracking rate of the central processor, measured in Megahertz (MHz) or Gigahertz (GHz). Higher baseline clock speeds indicate that more machine instruction loops can be processed per second.

Function: Accepts data and instructions from the user and converts them into a machine-readable form (binary code) for memory or the CPU.

Primary Examples: Keyboard, Mouse, Scanner.

2 Memory Unit Details ▼ Click to view

Function: The memory unit stores instructions and data. The processor reads instructions and reads/writes data from/to the memory during program runtime execution loops.

Memory Category Comparisons:

  • Registers: Smallest, ultra-fast temporary storage inside the CPU chip boundary.
  • Cache: Fast static memory buffer minimizing main RAM propagation lookup times.
  • RAM (Random Access Memory): Volatile read/write main memory acting as active processing workspace. Clears instantly at power shutdown.
  • ROM (Read Only Memory): Non-volatile memory holding static factory initialization routines (BIOS).
  • Secondary Memory: High-capacity, permanent, non-volatile auxiliary storage media (SSD/HDD, Pen Drives). Required because primary RAM cannot retain files or system files persistently when power is turned off.
3 Arithmetic & Logic Unit (ALU) Details ▼ Click to view

Function: The core computing engine room inside the CPU processor subsystem.

  • Arithmetic Operations: Addition, Subtraction, Multiplication, Division, Increments.
  • Logical Operations: Bitwise operations including AND, OR, NOT, XOR.
  • Comparison Operations: Evaluates data relationships like less than (<), greater than (>), or equal to (==).
  • Shift Operations: Moves bit sequences left (<<) or right (>>) inside data registers.
4 Control Unit (CU) Details ▼ Click to view

Function: The orchestrator and supervisor of the computing structure. It does not perform calculation loops itself, but controls instruction sequences.

Core CU Responsibilities:

  1. Fetches instructions sequentially from the primary memory space.
  2. Decodes instructions to decipher internal micro-operation configuration paths.
  3. Controls execution pathways inside the processor layout.
  4. Sends control signals across system blocks to synchronize input, output, and ALU processes.
5 Output Unit Details ▼ Click to view

Function: Produces computed computational results and outputs them back to the user environment.

Primary Examples: Monitor, Printer, Speaker.

The Three Main Bus Types:

1. Data Bus

Bi-directional wire group paths used to transfer active data parameters and opcode values across processing units.

2. Address Bus

Uni-directional line channels driven by the CPU to assign target coordinates of physical memory locations.

3. Control Bus

Carries synchronous control commands and status updates (Read/Write triggers, Interrupts) across system units.

The Instruction Cycle Phases

Also known as the Machine Cycle, this represents the explicit timeline sequence executed continuously by the processor:

1. Fetch Step

CPU copies the memory address from the Program Counter (PC) onto the Address Bus, then reads the instruction payload from memory into the Instruction Register (IR).

2. Decode Step

The Control Unit (CU) interprets the internal opcode fields within the IR to figure out what operation needs to be routed.

3. Execute Step

The Control Unit triggers hardware lines, routing tasks directly into the ALU logic arrays to perform calculation routines.
