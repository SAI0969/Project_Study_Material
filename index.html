Computer Architecture & Organization

1. Computer Architecture vs. Computer Organization

Here is a clear look at how the conceptual layout workspace contrasts with the physical design implementation layers:

Computer Architecture Computer Organization
Refers to what the computer does. Refers to how the computer does it.
It is the design and functionality visible to the programmer. It is the internal implementation of the hardware.
Examples: Instruction set, data types, addressing modes. Examples: Control unit, memory organization, ALU, buses.

In simple words:

  • Computer Architecture = What features the computer provides.
  • Computer Organization = How those features are built and work internally.
Easy way to remember:
Architecture = What  |  Organization = How

2. Fundamentals of Computer Architecture

Computer Architecture is the basic design and structure of a computer system. It defines what the computer can do and how the programmer interacts with it.

The main fundamentals are:

  • Instruction Set Architecture (ISA): The set of instructions a processor can execute.
  • Data Representation: How data such as numbers, characters, and symbols are stored in the computer.
  • Memory System: How the computer stores data and instructions (e.g., registers, cache, RAM).
  • Input and Output (I/O): How the computer communicates with peripheral hardware devices like keyboards and monitors.
  • Processor (CPU): The primary processing engine that executes instructions and handles mathematical operations.
  • Addressing Modes: The different algorithmic ways the CPU can calculate memory addresses to read or write data.

3. Organization of Von Neumann Machine

The Von Neumann Machine is a milestone framework layout configuration where both software instructions and operational raw data co-exist inside the same unified memory space.

Main Components

  • Memory Unit: Stores both program instructions and data.
  • Control Unit (CU): Controls and coordinates all operations of the computer. It handles fetching and decoding.
  • Arithmetic Logic Unit (ALU): Performs math calculations (+, -, etc.) and logical comparisons (AND, OR, NOT).
  • Input Unit: Accepts incoming operational instructions and data from the user.
  • Output Unit: Processes and renders target calculation outputs back to a view screen or file.

Working (Basic Steps)

  1. Input data and instructions are loaded into the system.
  2. The system stores them inside the unified memory framework.
  3. The CPU fetches the designated targeted instruction from memory.
  4. The Control Unit decodes the command parameters.
  5. The ALU executes the operation if arithmetic steps are needed.
  6. The final result is written back to memory or directed to an output system.

4. CPU Subsystems & Frameworks

CPU (Central Processing Unit)

  • The CPU operates as the main operational brain of the computer system.
  • It executes system instructions and manages processing pipelines.
  • It consists primary of the Control Unit (CU) and the Arithmetic Logic Unit (ALU).

Memory Systems

  • Memory acts as the staging unit that stores software data and program instructions.
  • The CPU reads data from memory blocks and commits calculation results back out to them.
  • Primary system layout models include standard RAM and high-speed local Cache lines.

Input-Output (I/O) Subsystems

  • Input hardware units send data vectors directly into the computer system workspace (e.g., Keyboards, Mice, Scanners).
  • Output hardware units present processed validation structures to users (e.g., Monitors, Printers).

Control Unit (CU) Operations

  • The Control Unit orchestrates and structures execution pipelines across the machine context.
  • It loads active software instructions straight out of volatile memory.
  • It parses and evaluates the operational signals.
  • It systematically directs the ALU, memory lines, and external I/O units to process actions.

5. Instruction Set Architecture (ISA) of a CPU – Registers

Instruction Set Architecture (ISA) is the set of instructions that a CPU can understand and execute. It defines how software communicates with the processor.

Registers

Registers are very small, very fast storage locations inside the CPU. They temporarily store data, instructions, and addresses while the CPU is processing them. Registers are faster than main memory (RAM).

Common Registers

  • Program Counter (PC): Stores the address of the next instruction to be executed.
  • Instruction Register (IR): Holds the current instruction being executed.
  • Accumulator (ACC): Stores intermediate arithmetic and logical results.
  • Memory Address Register (MAR): Holds the address of the memory location to be accessed.
  • Memory Data Register (MDR): Holds the data being read from or written to memory.

6. Instruction Execution Cycle

The Instruction Execution Cycle is the sequence of steps the CPU follows to execute a program instruction.

Steps

  1. 1. Fetch: The CPU fetches the instruction from memory. The instruction is stored in the Instruction Register (IR).
  2. 2. Decode: The Control Unit (CU) decodes the instruction. It determines what operation needs to be performed.
  3. 3. Execute: The CPU executes the instruction. The ALU performs arithmetic or logical operations if needed.
  4. 4. Store (Write Back): The result is stored in a register or memory.

7. RTL (Register Transfer Language) Interpretation of Instructions

Register Transfer Language (RTL) is a symbolic way of showing how data moves between registers, memory, and the CPU while executing an instruction. It describes register-level operations, not a programming language.

Example RTL Statements

Operation Target RTL Syntax Expression Description
Move data from Register R1 to Register R2 R2 ← R1 Copy the contents of R1 into R2.
Add two registers R3 ← R1 + R2 Add R1 and R2, store the result in R3.
Load data from memory R1 ← M[100] Load the data stored at memory address 100 into R1.
Store data into memory M[100] ← R1 Store the contents of R1 into memory address 100.

8. Addressing Modes

Addressing Modes are the different ways the CPU locates the data (operand) needed to execute an instruction.

Common Addressing Modes

  • 1. Immediate Addressing: The data is given directly in the instruction.
    Example: MOV R1, #10 (Here, 10 is the actual data).
  • 2. Direct Addressing: The instruction contains the memory address where the data is stored.
    Example: MOV R1, [100] (The CPU gets the data from memory location 100).
  • 3. Register Addressing: The data is stored in a CPU register.
    Example: MOV R1, R2 (Copy the contents of R2 to R1).
  • 4. Register Indirect Addressing: A register contains the memory address of the data.
    Example: MOV R1, [R2] (If R2 = 100, the CPU reads the data from memory location 100).
  • 5. Indexed Addressing: The memory address is obtained by adding an index register to a base address. Used for accessing arrays.
    Example: MOV R1, ARRAY[R2].

9. Types of Instructions

Instructions are commands given to the CPU to perform specific operations.

  • 1. Data Transfer Instructions: Used to move data between registers, memory, and I/O devices.
    Examples: MOV, LOAD, STORE
  • 2. Arithmetic Instructions: Used to perform mathematical operations.
    Examples: ADD, SUB, MUL, DIV
  • 3. Logical Instructions: Used to perform logical operations.
    Examples: AND, OR, NOT, XOR
  • 4. Branch (Control Transfer) Instructions: Used to change the normal sequence of program execution.
    Examples: JMP, CALL, RETURN
  • 5. Input/Output (I/O) Instructions: Used to read data from input devices or send data to output devices.
    Examples: IN, OUT
  • 6. Shift and Rotate Instructions: Used to shift or rotate bits of data.
    Examples: SHL, SHR, ROL, ROR

10. RISC vs CISC Architectural Models

Processor instruction set architectures generally fall into two broad design paradigms depending on whether they prioritize hardware simplicity or software compactness.

Feature RISC (Reduced Instruction Set Computer) CISC (Complex Instruction Set Computer)
Instruction Size Simple, fixed size (e.g., 32-bit). Complex, variable size (1 to 15 bytes).
Execution Time 1 clock cycle per instruction. Multiple clock cycles per instruction.
Memory Access LOAD/STORE architecture only (Operations happen in registers). Direct memory operands supported in arithmetic operations.
Register Count Large number of general-purpose registers. Fewer general-purpose registers.
Addressing Modes Few, simple addressing modes. Many, complex addressing modes.
Pipelining Highly optimized and easy to pipeline. Difficult to pipeline efficiently.
Examples ARM, MIPS, RISC-V, Apple Silicon (M-series). x86, x86-64 (Intel Core, AMD Ryzen).

11. Byte Ordering: Big-Endian vs Little-Endian

When multi-byte data types (like 32-bit integers) are stored in computer memory, Endianness defines the sequence or order in which individual bytes are stored at consecutive memory addresses.

Suppose we want to store the 32-bit hex value 0x12345678 starting at memory address 0x100:

Big-Endian

Stores the Most Significant Byte (MSB) at the smallest memory address.

120x100 (MSB)
340x101
560x102
780x103 (LSB)

Used in network protocols (Internet IP headers) and legacy mainframe systems.

Little-Endian

Stores the Least Significant Byte (LSB) at the smallest memory address.

780x100 (LSB)
560x101
340x102
120x103 (MSB)

Used by x86, x86-64 (Intel/AMD) processors and modern computer architectures.

Memory Tip:
Big-Endian: Big end (MSB) comes first in memory.
Little-Endian: Little end (LSB) comes first in memory.
