1. Introduction & Basics of Machine Learning

Artificial Intelligence vs. Machine Learning vs. Deep Learning

  • Artificial Intelligence (AI): Science of building intelligent systems capable of performing tasks that normally require human intelligence (e.g., Speech Recognition, Computer Vision, Robotics, NLP, Autonomous Vehicles, Expert Systems).
  • Machine Learning (ML): A subset of AI focused on statistical algorithms that learn patterns from data automatically without being explicitly programmed.
  • Deep Learning (DL): A subset of ML utilizing multi-layer artificial neural networks to extract feature hierarchies from complex data.

Definitions of Machine Learning

  • Arthur Samuel (1959): "Field of study that gives computers the ability to learn without being explicitly programmed."
  • Tom Mitchell: "A computer program learns from experience E with respect to some task T and performance measure P if its performance at task T, measured by P, improves with experience E." Formally: Learning = f(E, T, P).

Traditional Programming vs. Machine Learning

Traditional Programming Machine Learning
Input + Program (Rules) → Output Input + Output → Learning (Model Rules)
Requires explicit hardcoded rules for every scenario. Discovers hidden patterns and automatically adapts with new data.

General Machine Learning Workflow

Standard Pipeline:
Problem Definition → Data Collection → Preprocessing → Training → Testing → Deployment

2. Types of Machine Learning

Supervised Learning

Operates on training datasets with target labels: D = {(x_i, y_i)}. The objective is learning a mapping function f(x) → y.

  • Regression: Predicts continuous numerical values (e.g., House Price Prediction, Temperature, Stock Prices, Sales Forecasting). Evaluated using Mean Squared Error: MSE = (1/n) ∑ (y_i - &ŷ;_i)².
  • Classification: Predicts discrete categorical class labels (e.g., Spam/Not Spam, Cancer/Healthy, Pass/Fail, Credit Risk). Evaluated using Accuracy and Confusion Matrices.

Unsupervised Learning

Operates on input features without target labels: D = {x_1, x_2, ..., x_n}. Focuses on discovering underlying patterns and structures.

  • Clustering: Grouping similar data points (e.g., Customer Segmentation, K-Means, Agglomerative Hierarchical).
  • Association Rule Mining: Identifying frequent item relationships (e.g., Market Basket Analysis).
  • Dimensionality Reduction: Projecting data into lower dimensional spaces (e.g., PCA, Image Compression).

Semi-Supervised Learning

Combines a small amount of labeled data (D_L) with a large amount of unlabeled data (D_U): D = D_L + D_U. Highly effective in domains where labeling is expensive (e.g., Medical Imaging, Speech Recognition, Text Classification).

Reinforcement Learning

An Agent interacts sequentially with an Environment by selecting actions and receiving state updates and numeric Rewards or penalties. Objective: Maximize total discounted reward over time.

3. Machine Learning Challenges

  1. Insufficient Training Data: Small sample sizes prevent algorithms from establishing statistically robust pattern representations[cite: 1].
  2. Poor Quality Data: Duplicate records, incorrect labels, and inconsistent measurements lead to "Garbage In, Garbage Out"[cite: 1].
  3. Missing Values: Null entries requiring removal or statistical imputation (Mean, Median, Mode, KNN)[cite: 1].
  4. Noisy Data: Systemic errors, sensor glitches, or human typing mistakes represented as y = x + ε[cite: 1].
  5. High Dimensional Data: High feature dimensions (e.g., d = 1000) cause data sparsity and computation bottlenecks (Curse of Dimensionality)[cite: 1].
  6. Imbalanced Datasets: Severe class distribution skews that bias classification decision boundaries[cite: 1].
  7. Overfitting: Model memorizes training noise rather than general trends; yields low training error but high test error (High Variance)[cite: 1].
  8. Underfitting: Model is too simplistic to capture dataset patterns; yields high training and high test error (High Bias)[cite: 1].
  9. Bias-Variance Tradeoff: Total Prediction Error = Bias² + Variance + Noise[cite: 1].
  10. Computational Complexity: Resource bottlenecks when scaling training pipelines over massive infrastructure[cite: 1].

4. Testing, Validation & Evaluation Metrics

Data Splitting

  • Training Set: Used to fit model parameters (θ). Typical splits: 70-15-15% or 80-20%.
  • Validation Set: Used for hyperparameter tuning (learning rate, depth, trees), model selection, and early stopping.
  • Testing Set: Evaluates final generalization performance (evaluated only once at the end).

Validation Methods

  • Hold-Out Validation: Simple train/test split. Fast but sensitive to data partitioning variance.
  • K-Fold Cross-Validation: Splits data into K equal folds (e.g., K=5 or 10). Trains on K-1 folds, tests on 1 fold iteratively. Overall score is the mean across folds.
  • Leave-One-Out Cross-Validation (LOOCV): K-Fold where K = N (sample count). Low bias, but computationally expensive.

10 Guidelines for Machine Learning Experiments

  1. Clearly define the problem.
  2. Collect representative data.
  3. Clean and preprocess data.
  4. Split data correctly.
  5. Select suitable algorithms.
  6. Tune hyperparameters.
  7. Evaluate using multiple metrics.
  8. Compare with baseline models.
  9. Document experiments.
  10. Ensure reproducibility (e.g., fixing random seeds).

5. Data Preparation

Data Cleaning & Preprocessing

  • Objectives: Remove duplicates, handle missing values, filter noise, correct data types.
  • Handling missing values: Delete records, Mean Imputation, Median Imputation, Mode Imputation, or KNN Imputation.

Feature Scaling

  • Normalization (Min-Max Scaling): Rescales data into range [0, 1]. Formula: x' = (x - x_min) / (x_max - x_min). Ideal for Neural Networks.
  • Standardization (Z-score): Rescales data to mean = 0 and standard deviation = 1. Formula: z = (x - μ) / σ. Ideal for Logistic Regression, SVM, and distance-based methods.

Feature Engineering & Selection

  • Feature Engineering: Creating new variables (e.g., BMI = Weight / Height², age groups, date parsing, polynomial features).
  • Feature Selection: Removing irrelevant/redundant variables via Pearson Correlation, Chi-Square tests, Mutual Information, and Recursive Feature Elimination (RFE).
  • Pearson Correlation Coefficient (r): Measures linear correlation bounded in [-1, 1].

6. Data Visualization Techniques

Key Plotting Methods

  • Histograms (plt.hist): Displays frequency distribution of continuous numerical data partitioned into bins.
  • Bar Plots (plt.bar): Compares discrete categorical values or metric group summaries.
  • Box Plots (plt.boxplot): Displays 5-number summaries (Min, Q1, Median, Q3, Max) and identifies outliers.
  • Scatter Plots (plt.scatter): Plots individual points along 2 continuous feature axes to observe correlations.
  • Trend Lines: Best-fit regression lines overlaid on scatter plots (e.g., using np.polyfit(x, y, 1)).
